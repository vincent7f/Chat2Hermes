# v0.4.2 实施计划

本文档记录已确认的 v0.4.2 UI/UX 改进范围、实施顺序与验收标准。  
状态约定：`[ ]` 待开始，`[~]` 进行中，`[x]` 已完成。

## 1. 目标

- 降低首次上手成本，提升主流程可理解性。
- 提升高频操作效率（发送、恢复、切换 profile）。
- 降低误操作与状态不确定感（刷新/续拉/失败场景）。

## 2. 范围与优先级

### P0（高优先）

- [ ] A1 消息状态可视化统一（接收中/待恢复/已恢复/失败）
- [ ] A2 长按菜单重排分组（会话控制、内容操作、其他）
- [ ] A3 输入区可用性增强（发送状态反馈与提示优化）

### P1（中优先）

- [ ] B4 设置页信息分区（连接配置、模型与鉴权、稳定性）
- [ ] B5 Profile 操作反馈补强（切换/删除后的明确提示）
- [ ] B6 术语统一（刷新/继续接收/重发等口径）

### P2（体验增强）

- [ ] C7 微交互一致性（菜单、折展、状态切换动画）
- [ ] C8 无障碍与触达优化（点击区域、语义、对比度）
- [ ] C9 空状态与异常态增强（给出下一步行动）

## 3. 逐项实施顺序

按以下固定顺序执行并逐项确认：

1. A1 -> 2) A2 -> 3) A3 -> 4) B4 -> 5) B5 -> 6) B6 -> 7) C7 -> 8) C8 -> 9) C9

每项流程：

1) 设计说明（变更点 + 影响文件）  
2) 代码实现  
3) 必要验证（lint / 关键流程自测）  
4) 记录结果（本文件勾选并补充简述）

## 4. 各项验收标准

### A1 消息状态可视化统一

- 助手消息可直观看到当前状态。
- 0 字符可恢复消息可见“待恢复”并可触发刷新/继续接收。
- 状态切换无误导（已恢复 -> 接收中 -> 成功/失败）。

### A2 长按菜单重排分组

- 菜单分组顺序稳定、主次清晰。
- 不可恢复时隐藏“刷新”。
- 现有功能不回归（复制、选择、朗读、重发等）。

### A3 输入区可用性增强

- 发送按钮状态变化明确。
- 输入区提示文案与行为一致（不改变核心发送逻辑）。

### B4 设置页信息分区

- 设置项按主题分区，结构清晰。
- 配置读写逻辑保持兼容。

### B5 Profile 操作反馈补强

- 切换/删除 profile 后有即时反馈。
- 删除后目标 profile 明确可知。

### B6 术语统一

- “刷新/继续接收/重发”定义统一并在界面一致使用。

### C7 微交互一致性

- 动画节奏统一且不过度影响操作效率。

### C8 无障碍与触达优化

- 关键控件可达性、语义标签、对比度满足基本标准。

### C9 空状态与异常态增强

- 无消息、断线、失败状态均有明确下一步引导。
