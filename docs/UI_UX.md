# UI / UX — Herdroid

本文档约定信息架构、视觉与无障碍，对齐 Material Design 3 与 [PRD.md](PRD.md) 中的易用性要求。

## 1. 设计系统

- **Material 3**：动态配色（Monet）在支持的系统上启用；提供浅色/深色主题与手动切换入口（可放在设置）。
- **排版与间距**：遵循 M3 类型刻度与 4dp/8dp 网格；避免主界面信息过载。
- **图标**：使用 Material Symbols 或等价矢量资源，保证清晰度。

## 2. 信息架构

| 区域 | 内容 |
|------|------|
| 主界面 | 连接状态（已连接 / 连接中 / 断开 / 错误）、对话消息列表与输入 |
| 设置 | HA：协议、地址、端口；OpenAI 兼容 API：API Key、模型名（根地址由协议/地址/端口拼接）；关于与开源许可（后续） |

导航：**底部导航**或 **单页 + 顶部「设置」图标** 二选一；MVP 推荐 **主界面 + 设置页** 两级即可。

## 3. 状态与反馈

- **连接中**：显示不确定进度或骨架，避免空白。
- **错误**：一句人话说明 + 可选「重试」；技术细节折叠或仅 debug。
- **空状态**：首次进入无消息时，简短引导去设置配置 HA。

## 4. 无障碍（a11y）

- **TalkBack**：所有可点击控件带 `contentDescription`；开关状态可被朗读。
- **字体缩放**：使用 `sp`；布局在最大字体下不截断关键控件（可滚动）。
- **对比度**：文本与背景对比度满足 WCAG 在 M3 下的常规实践。

## 5. 文案原则

- 短句、动词开头（如「连接」「重试」「清空对话」）。
- 与 [PRIVACY_AND_SECURITY.md](PRIVACY_AND_SECURITY.md) 中数据出境提示一致，不恐吓用户。

## 6. 与架构的对应

- `UiState` 为不可变数据类，包含连接枚举、消息列表、加载状态、错误信息。
- 主题与导航由 `app` 模块顶层 Composable 统一提供。
