# 隐私与安全 — Herdroid

本文档说明应用处理的数据范围、局域网与对话 API 风险及日志策略，与 [PRD.md](PRD.md)、[ANDROID_CONVENTIONS.md](ANDROID_CONVENTIONS.md) 一致。

## 1. 数据范围

| 数据类型 | 存储位置 | 说明 |
|----------|----------|------|
| HA 地址与端口、协议 | 设备本地（DataStore） | 不上传至第三方 |
| 服务基础 URL、API Key、模型名 | 设备本地 | API Key 等敏感项仅存本地，不明文写入版本库 |
| HA 返回消息内容 | 内存为主；若未来做历史记录，默认本地数据库并加密可选 | PRD 未要求云同步则不做 |
| 崩溃与诊断日志 | 若接入上报工具，遵循下方第 4 节 | 默认最小化 |

## 2. 局域网连接（HA）

- 流量仅在**用户配置的**局域网或用户路由可达的地址上传输；应用不扫描用户通讯录或无关设备（除非未来功能经单独 PRD 批准）。
- 若 HA 使用 HTTP：用户需知悉同网络内嗅探风险；HTTPS 推荐用于敏感对话。

## 3. OpenAI 兼容对话 API

- 用户发送的**对话文本**会到达用户配置的 HTTP(S) 服务端；须在设置中明确由用户掌控端点。
- 建议使用 **HTTPS**；证书校验遵循系统信任库或应用内配置的固定证书策略（与网络安全配置一致）。

## 4. 日志与崩溃报告

- **发布版**：不记录完整 HA 消息正文到持久日志；调试日志用 `BuildConfig.DEBUG` 或等价门控。
- 若集成 Firebase Crashlytics / Sentry 等：关闭或过滤可能含用户对话的字段；隐私政策链接在应用上架前补充。

## 5. 权限与攻击面

- 仅申请实现 PRD 所需权限；见 ANDROID_CONVENTIONS 清单节。
- 不动态加载不可信代码；依赖库定期审计（Dependabot 等）。

## 6. 合规与文档

- 面向 Google Play 上架时，补充「数据安全」表单与隐私政策 URL。
- 本文档随功能扩展更新。
